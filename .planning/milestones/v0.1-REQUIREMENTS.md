# Requirements Archive: framershot v0.1

**Archived:** 2026-05-22 (milestone v0.1 shipped)
**Defined:** 2026-05-19
**Core Value:** Reliably capture clean, retina-quality screenshots of Framer sites without ghosted navs, half-played animations, or missing lazy-loaded content.

## v1 Requirements (all delivered)

Scope: v0.1 minimum viable — CLI + config + single page/viewport full-page stitched capture with prepare pipeline.

### CLI

- [x] **CLI-01**: User can run `framershot capture <config.yaml>` from any directory and produce screenshots — *delivered Phase 2 (validated end-to-end via Phase 5 stitched output)*
- [x] **CLI-02**: User sees terminal progress output (current page, current step) while capture runs — *delivered Phase 6 via ora spinner across 7 step boundaries*
- [x] **CLI-03**: User sees actionable error messages when capture fails (config error, network error, missing selector) — *delivered Phase 6 via `formatError` dispatcher (ConfigError / BrowserError / TimeoutError / default)*

### Config

- [x] **CFG-01**: User can author a YAML config with name, baseUrl, output template, deviceScaleFactor, single viewport, single page, and prepare options — *delivered Phase 2*
- [x] **CFG-02**: Invalid config files are rejected with specific error messages pointing to the bad field (zod-based validation) — *delivered Phase 2 via `formatZodError` in ConfigError*
- [x] **CFG-03**: Output path template supports `{date}`, `{viewport}`, `{page}` placeholders — *delivered Phase 2 via `resolveTemplate`*

### Capture

- [x] **CAP-01**: Browser launches Chromium headless with configured viewport width and height — *delivered Phase 3*
- [x] **CAP-02**: Capture honors `deviceScaleFactor` for retina-quality output (2x or 3x) — *delivered Phase 3*
- [x] **CAP-03**: Browser navigates to `baseUrl` + page path and waits for `networkidle` before proceeding — *delivered Phase 3 (15s nav timeout per RESEARCH §Pitfall 2)*
- [x] **CAP-04**: Capture waits for `document.fonts.ready` so screenshots don't use fallback fonts — *delivered Phase 3 via `waitForFunction`*

### Prepare

- [x] **PREP-01**: Animations are disabled before capture via CSS injection (`animation: none !important; transition: none !important`) — *delivered Phase 4*
- [x] **PREP-02**: Framer Motion appear effects fire instantly via IntersectionObserver replacement so in-view triggers complete before capture — *delivered Phase 4 (canonical hook — pre-nav context-level shim)*
- [x] **PREP-03**: Selectors in the config's `hide` list are hidden from the page before capture — *delivered Phase 4 via `visibility: hidden !important` (preserves layout heights)*
- [x] **PREP-04**: Scroll prime runs (scroll to bottom in viewport-height steps with wait, then scroll back to top) when enabled in config — *delivered Phase 4 (200ms inter-step waits, instant-only scroll)*
- [x] **PREP-05**: Configured `extraDelay` (ms) is honored before capture begins — *delivered Phase 4*

### Output

- [x] **OUT-01**: Full-page screenshot is captured by scrolling in viewport-height steps and capturing each frame — *delivered Phase 5 (captureFrames with rAF wait + last-frame overlap clamp)*
- [x] **OUT-02**: Captured frames are stitched into a single PNG using sharp with no ghosted sticky elements and no visible seams — *delivered Phase 5 (stitchFrames sharp composite)*
- [x] **OUT-03**: Output file is written to the templated path with parent directories created as needed — *delivered Phase 5 (mkdir recursive + writeFile in captureFullPage orchestrator)*

## v2 Requirements (carried forward to future milestones)

### Multi-target capture (v0.2)

- **MULTI-01**: User can declare multiple viewports in config and capture all in one run
- **MULTI-02**: User can declare multiple pages in config and capture all in one run
- **MULTI-03**: User can pass `--viewport=<name>` to restrict a run to one viewport
- **MULTI-04**: User can pass `--pages=<list>` to restrict a run to specific pages

### Region capture (v0.2)

- **REGION-01**: User can declare named regions by CSS selector and capture only that element with padding
- **REGION-02**: User can declare named regions by `from`/`to` anchors and capture the bounding box between them
- **REGION-03**: User can pass `--only=<region>` to capture a specific region instead of the full page

### Advanced (v0.3)

- **DIFF-01**: User can run a diff command comparing two captures and get a diff image
- **HOOK-01**: User can declare pre-capture hooks (scripts) that run before capture to set cookies or accept consent
- **AUTH-01**: User can configure HTTP basic auth or password-protected Framer staging URL credentials

## Out of Scope (unchanged)

| Feature | Reason |
|---------|--------|
| Visual regression testing as a product | Capture is the goal; diff mode is a tool, not the north star |
| npm publish + friendly stranger UX + comprehensive README | Personal tool; polish only what makes daily use frictionless |
| Cross-browser capture (Firefox, WebKit) | Chromium covers the use case; smaller install |
| Non-Framer site optimizations / general-purpose tool | Framer-specific quirks are why this exists; generalizing dilutes focus |
| Cloud / hosted capture service | Local-only by design; no server or infrastructure |
| Headed mode as default | Speed and reproducibility favor headless; headed only as debug flag if added |
