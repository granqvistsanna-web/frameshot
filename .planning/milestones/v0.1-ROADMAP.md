# Milestone v0.1: MVP

**Status:** SHIPPED 2026-05-22
**Phases:** 1–6
**Total Plans:** 16

## Overview

v0.1 delivers a working local CLI (`framershot capture <config.yaml>`) that captures clean, retina-quality full-page screenshots of Framer sites. Six phases, shipped over four days (2026-05-19 → 2026-05-22), build the tool from scratch: project scaffold, CLI + config loading, browser launch and navigation, the prepare pipeline (the differentiating work — Framer Motion surgical disable via IntersectionObserver replacement + scroll prime), the scroll-and-stitch capture loop, and terminal UX polish (ora progress + actionable errors).

## Milestone Summary

**Decimal Phases:** None — all phases shipped in original numeric order without urgent insertions.

**Key Decisions:**

- Framer Motion is disabled via IntersectionObserver replacement (the canonical hook), NOT a fabricated `window.__framer_motion_disabled` global. Decision made in 04-RESEARCH; corrected a plan-time misconception before implementation.
- Element hiding uses `visibility: hidden !important`, NOT `display: none` — preserves layout heights so Phase 5's viewport-step scroll math stays valid.
- Capture uses scroll-and-stitch (per-viewport `page.screenshot()` + sharp composite), NOT Playwright's native `fullPage: true` — `fullPage` ghosts sticky elements on every viewport boundary; our approach captures sticky once at the top.
- `playwright-chromium` only (not full `playwright`) — Chromium-only by PROJECT scope; smaller install.
- Single error sink: library modules throw typed errors (`ConfigError`, `BrowserError`); only `index.js` top-level catch calls `console.error` / `process.exit`. CLI formats and exits; libs stay pure.
- ESM + executable shebang committed via `git update-index --chmod=+x` so `npm link` produces a working CLI immediately after clone.
- Caret-major version ranges in `package.json` (`^1`, `^12`, etc.) — let npm resolve latest within major rather than over-pinning.
- Wave-based parallel plan execution (Phase 4 wave 1 had 4 parallel plans; Phase 5 wave 1 had 2) — zero `files_modified` overlap kept merges trivial.

**Issues Resolved:**

- Playwright's `fullPage: true` ghost-stickies problem — solved by scroll-and-stitch with sharp.
- Framer Motion `whileInView` triggers not firing in headless capture — solved by IntersectionObserver shim that fires synchronously.
- Lazy-loaded images blank below the fold — solved by `scrollPrime` (full scroll cycle before capture).
- Font-flash in screenshots — solved by `await document.fonts.ready` after navigation.
- Raw zod dumps in error output — solved by `formatZodError` in `ConfigError` + dispatcher in `formatError`.

**Issues Deferred:**

- Multi-viewport per run (MULTI-01, deferred to v0.2)
- Multi-page per config (MULTI-02, deferred to v0.2)
- CLI flag filters `--viewport=`, `--pages=`, `--only=` (MULTI-03, MULTI-04, REGION-03; deferred to v0.2)
- Region capture by CSS selector / from-to anchors (REGION-01, REGION-02; deferred to v0.2)
- Diff mode (DIFF-01; deferred to v0.3)
- Pre-capture hooks for cookies/consent (HOOK-01; deferred to v0.3)
- Auth for password-protected Framer staging URLs (AUTH-01; deferred to v0.3)

**Technical Debt Incurred:**

- `src/cli.js` carries some Phase 3 dynamic `import()` of `node:fs/promises` / `node:path` in the smoke branch — never lifted to top-level imports. Low priority; only affects smoke path.
- `hideSummary` returned by `hideSelectors` is captured in CLI but only surfaced as a warning in Phase 6; no structured "missed selector" channel for programmatic consumers (none exist yet — fine for now).
- No unit tests — verification is via the hermetic smoke fixture (`samples/serve-smoke.js` + `samples/smoke.yaml`) end-to-end. Adequate for personal-tool scope.
