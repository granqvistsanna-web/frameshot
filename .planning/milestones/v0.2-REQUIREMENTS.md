**Carry-forward context:** v0.1 shipped the full prepare pipeline, scroll-stitch full-page capture, ora progress, and typed-error reporting. v0.2 added breadth without compromising that pipeline — multi-viewport iteration and region capture both reuse the existing prepare → capture flow.

## v0.2 Requirements (all validated)

### Multi-viewport

- [x] **MULTI-01**: User can declare multiple viewports in one config (array of `{ name, width, height, deviceScaleFactor }`) and one `framershot capture` run produces one full-page PNG per viewport. Output template `{viewport}` placeholder resolves to the per-viewport `name`. Each viewport gets its own browser context (no shared state). — **Shipped Phase 7 (verified 2026-05-24, 5/5 success criteria PASS)**

### Region capture

- [x] **REGION-01**: User can declare named regions in config by CSS selector (`regions: [{ name, selector, padding? }]`) and capture only that element to its own PNG — scrolled into view, prepare pipeline applied, padding honored. — **Shipped Phase 8 (verified 2026-05-24, SC #1 PASS)**

- [x] **REGION-02**: User can declare named regions by from/to anchors (`regions: [{ name, from, to, padding? }]`) — capture computes the bounding box between the two anchor elements and clips to it. — **Shipped Phase 8 (verified 2026-05-24, SC #2 PASS)**

- [x] **REGION-03**: User can pass `--only=<region-name>` to `framershot capture` to capture a single named region instead of the full page. Without the flag, the full-page stitch behavior from v0.1 is unchanged. — **Shipped Phase 8 (verified 2026-05-24, SC #3 PASS)**

## Validated outcomes

- **MULTI-01 — Validated as specified.** `viewports: [...]` array shape works; `{viewport}` placeholder slugifies to distinct paths; per-viewport browser context isolation observable in live runs (`.does-not-exist` selector warning fires once per viewport — proof of fresh prepare-pipeline per iteration). v0.1 singular `viewport:` shape preserved via root `.superRefine` mutual-exclusivity + normalize `.transform` in `src/config/schema.js`.

- **REGION-01 — Validated as specified.** `captureRegion` selector mode applies the full prepare pipeline (CSS animation disable + IO shim + selector hiding + scroll prime) before screenshot; padding honored via `padRect` → `clampToDocument` geometry helpers.

- **REGION-02 — Validated as specified.** Anchor mode computes `unionRect` between resolved `from` and `to` anchor bboxes, applies padding, clamps to document bounds, screenshots via `page.screenshot({ clip })`.

- **REGION-03 — Validated as specified.** `--only=<region>` validates upfront (fail-fast, pre-Chromium-launch); when valid, skips full-page stitch and all other regions; when omitted, full-page stitch + all regions run (Open-Q#1 lock A: regions are additive, not replacing). `--smoke`/`--only` mutex throws `RegionError` via Guard 4 of `formatError` dispatcher.

## Out of Scope (reasoning still valid post-v0.2)

| Feature | Reason |
|---------|--------|
| Visual regression testing as a product | Capture is the goal; diff is a tool |
| npm publish + stranger-friendly UX | Personal tool |
| Cross-browser capture (Firefox, WebKit) | Chromium covers the use case |
| Non-Framer site optimizations | Framer-specific quirks are why this exists |
| Cloud / hosted capture service | Local-only by design |
| Headed mode as default | Speed and reproducibility favor headless |
| Unit test suite | Verified via hermetic smoke fixtures (`samples/smoke.yaml`, `samples/smoke-multi.yaml`, `samples/smoke-regions.yaml`); adequate for personal-tool scope |
