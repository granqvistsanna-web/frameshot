<objective>
Lock the v0.2 config shape at the schema/loader boundary so every downstream consumer (runCapture, launcher, cli, server) reads ONE normalized shape: `config.viewports` as an array of `{ name, width, height }` with `name` required and unique across the array.

Purpose: Per CONTEXT.md D-01 + D-02 + the "Specifics" section, push complexity into the schema/loader and keep the rest of the code branch-free. Schema-enforced uniqueness is the only guarantee that the `{viewport}` placeholder in the output template (ROADMAP success criterion #2) prevents PNG overwrites.

Output: Extended `src/config/schema.js` (new `viewportEntrySchema` + `viewportsSchema` + root refinement + normalize) and updated `src/config/load.js` JSDoc; v0.1 singular configs (`samples/smoke.yaml`, `samples/sample.yaml`) still validate; new plural configs validate; conflicting/missing/duplicate configurations fail with clean field-named ConfigError messages.
</objective>
