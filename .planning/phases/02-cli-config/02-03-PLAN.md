  <action>
    Create `/Users/sannagranqvist/Documents/App/screenshotter/src/output/template.js` as an ESM module with exactly one named export: `resolveTemplate`.

    **Imports:** none. This is a pure function with zero dependencies.

    **Signature:** `export function resolveTemplate(template, { date, viewport, page })`
      - `template` (string): the output template from the config, e.g. `./screenshots/{date}/{viewport}/{page}.png`.
      - `date` (string): a pre-formatted date string. The caller passes `new Date().toISOString().slice(0, 10)` — keeps this function pure and deterministic.
      - `viewport` (string): viewport name from the config; slugified internally.
      - `page` (string): page name (or slug derived from page path) from the config; slugified internally.

    **Implementation shape (RESEARCH.md §Pattern 4 verbatim — ~10 lines total):**
      - Define a private `slugify(value)` helper: lowercase the input, replace runs of `[^a-zA-Z0-9._-]` with `-`, trim leading/trailing dashes. Preserves `.`, `-`, `_`.
      - The exported function applies three `.replaceAll(...)` calls in sequence: `{date}` → date (no slug), `{viewport}` → `slugify(viewport)`, `{page}` → `slugify(page)`.
      - Return the resulting string.

    **Behavior rules (RESEARCH.md §Pattern 4 — these are deliberate, not optional):**
      - `{date}` is NOT slugified. The hyphens in `YYYY-MM-DD` are intentional and the input is already path-safe.
      - The template itself is NOT slugified. `/` path separators must survive.
      - `{viewport}` and `{page}` ARE slugified.
      - Unknown placeholders like `{Date}` (wrong case) or `{viewprt}` (typo) are LEFT LITERAL in the output. RESEARCH.md §Pattern 4 "Unknown placeholder behavior" explains: erroring would require an allowlist that must grow with v0.2; literal pass-through surfaces typos visibly in the resulting path.
      - No `new Date()` inside the function. The caller computes the date string.
      - No path-traversal sanitization. Trust boundary: the config author is the user (PROJECT.md: personal tool). Phase 5's writer is the right place to add hardening if/when needed.
      - No throw on missing context values. If `viewport` is `undefined`, `slugify(undefined)` returns the literal string `"undefined"` — visible and noisy is the right v0.1 failure mode.

    **Anti-patterns to avoid:**
      - Do NOT use a single regex with a callback (`/\{(date|viewport|page)\}/g`). Three sequential `.replaceAll` calls are simpler and the form RESEARCH.md §Pattern 4 documents.
      - Do NOT add a `date` default (`date ?? new Date()...`). Force the caller to be explicit; keeps the function pure.
      - Do NOT import `node:path`. The template author wrote the slashes; trust them.
      - Do NOT add a "format" or "strftime" abstraction. Three hardcoded placeholders is the spec.
