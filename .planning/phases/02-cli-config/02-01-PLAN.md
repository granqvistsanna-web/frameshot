## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `process.argv` → commander | Untrusted strings from the user's shell cross into commander's parser. commander v14 (~140M weekly downloads, vetted in Phase 1) handles malformed argv safely. |
| config path argument → filesystem | The string after `framershot capture` is passed to `loadConfig`, which calls `path.resolve` + `fs.readFile`. PROJECT.md frames the user as the threat boundary — they can read any file they own; that's expected behavior, not a vulnerability. |
