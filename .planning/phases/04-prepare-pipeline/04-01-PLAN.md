<objective>
Build the pre-navigation animation-guards module — a single async function that, given a BrowserContext, registers two init scripts that fire on every new document in that context. Init script 1 replaces the global IntersectionObserver constructor with a shim that immediately reports every observed target as intersecting (so Framer Motion's `whileInView` callbacks complete on the first observe() call). Init script 2 injects a universal CSS rule that disables `animation` and `transition` properties site-wide. Both guards are gated by the single `prepareConfig.animations` boolean — when false, the function short-circuits without registering either.

Purpose: PREP-01 (CSS animation/transition freeze) and PREP-02 (Framer Motion in-view trigger via IO replacement) are NOT independently shippable — RESEARCH.md §Pitfall 12 documents that shimming IO without freezing CSS leaves Framer's WAAPI tweens free to play (elements arrive at the right state then animate back to it), and freezing CSS without shimming IO leaves Framer elements stuck at their `initial: { opacity: 0 }` state forever. The two guards share the same lifecycle (must register BEFORE the first Page is created in the context — RESEARCH.md §Pitfall 3 / Pattern 1) and the same config gate (a single boolean). Co-locating them in one module under one config flag matches the schema's intent (schema.js:25 — single `animations: boolean`) and prevents an undefined intermediate state where one guard is on and the other off. The roadmap's per-requirement plan boundaries (04-01 = PREP-01, 04-02 = PREP-02) are reconciled here in favor of the research's lifecycle-aware split: PREP-01 + PREP-02 ship together in this plan; PREP-03 stays in 04-02; PREP-04 + PREP-05 stay in 04-03; orchestrator + CLI wiring move to a NEW 04-05 (wave 2). 04-04 is the optional ALL-IN-ONE post-nav plan if reorganization needs collapsing — see this phase's plan headers for the final split.

Output: `src/prepare/animations.js` — one named async export, two `await context.addInitScript(...)` calls, no I/O beyond the BrowserContext argument, no console output.

      Step 1 — Guard 1: IntersectionObserver shim via `context.addInitScript` (PREP-02 — RESEARCH.md §Pattern 1 lines 279-312 verbatim usable). The inline callback executes in the page context on every new document.

        ```
        await context.addInitScript(() => {
          window.__originalIO = window.IntersectionObserver;
          const ShimIO = class {
            constructor(callback, options) {
              this._callback = callback;
              this._options = options;
              this._targets = new Set();
            }
            observe(target) {
              this._targets.add(target);
              // Fire on next microtask so the consumer's observe() call returns
              // before the callback executes — matches native semantics. Native
              // IO never fires sync from observe(); it batches to the next frame.
              // Microtask is "soon enough" for Framer Motion's purposes.
              Promise.resolve().then(() => {
                const rect = target.getBoundingClientRect?.() ?? {};
                const entry = {
                  target,
                  isIntersecting: true,
                  intersectionRatio: 1,
                  boundingClientRect: rect,
                  intersectionRect: rect,
                  rootBounds: null,
                  time: performance.now(),
                };
                this._callback([entry], this);
              });
            }
            unobserve(target) { this._targets.delete(target); }
            disconnect() { this._targets.clear(); }
            takeRecords() { return []; }
          };
          window.IntersectionObserver = ShimIO;
          // Belt-and-braces: some bundlers reference globalThis.IntersectionObserver
          // separately from window.IntersectionObserver in browser environments
          // they're the same binding, but explicit assignment guards against
          // any module that grabbed `globalThis` first. 04-PATTERNS §animations.js
          // body shape (Guard 1 — and globalThis.IntersectionObserver per Pitfall 11).
          globalThis.IntersectionObserver = ShimIO;
        });
        ```

        Notes (do NOT inline as comments — keep the source readable; these are anchors for the executor's mental model):
        - The shim is INTENTIONALLY incorrect by polyfill standards — it LIES (reports `isIntersecting: true` immediately for every target). RESEARCH §Alternatives Considered explains why a real polyfill is the wrong tool.
        - The microtask scheduling via `Promise.resolve().then(...)` matches native batching semantics enough for Framer Motion. Synchronous firing from `observe()` would violate the IO contract that `observe()` returns before the first callback (some downstream code can assume non-reentrancy here).
        - `window.__originalIO` is preserved as a debug aid (a headed debug session can read it to compare against the shim).
        - The class includes ALL five methods (`observe`, `unobserve`, `disconnect`, `takeRecords`) plus the constructor — partial implementations would break libraries that call `disconnect` during cleanup.
        - Do NOT name the static `root`, `rootMargin`, `thresholds` instance properties — RESEARCH §Pitfall 1 noted some libraries feature-detect via these, but Framer doesn't, and adding them is YAGNI. If a future Framer version requires them, the shim can be extended.

      Step 2 — Guard 2: CSS injection via `context.addInitScript` (PREP-01 — RESEARCH.md §Pattern 1 lines 328-359 verbatim usable). The inline callback inserts a `<style>` element into the document.

        ```
        await context.addInitScript(() => {
          const css = `
            *, *::before, *::after {
              animation: none !important;
              transition: none !important;
              animation-delay: 0s !important;
              animation-duration: 0s !important;
              transition-delay: 0s !important;
              transition-duration: 0s !important;
            }
          `;
          const inject = () => {
            if (document.querySelector('style[data-framershot="animation-guards"]')) return;
            const style = document.createElement('style');
            style.dataset.framershot = 'animation-guards';
            style.textContent = css;
            (document.head ?? document.documentElement).appendChild(style);
          };
          if (document.head) {
            inject();
          } else {
            // addInitScript fires at document_start when document.documentElement
            // exists but document.head may not yet. readystatechange catches
            // the head's arrival at the earliest reliable point.
            document.addEventListener('readystatechange', () => {
              if (document.head) inject();
            });
          }
        });
        ```
