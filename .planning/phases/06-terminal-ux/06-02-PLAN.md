<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Error thrown anywhere → formatError → console.error → terminal | All formatted output is presentation-only with no remote audience; user IS the only consumer (PROJECT.md personal-tool posture). |
| Error → chalk wrapping → ANSI escape codes on stderr | chalk wraps user-supplied strings (like err.message which may contain a YAML-supplied baseUrl) as function arguments; chalk does NOT interpret input as ANSI codes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-V7-01 | Information Disclosure | Stack traces in default-branch `formatError` reveal absolute file paths from the user's machine | accept | Local CLI, user IS the only audience (PROJECT.md). Stack traces are dimmed via chalk.dim so they're available for debugging but visually subordinated. ASVS V7 recommends not leaking stack traces to REMOTE audiences — there are no remote audiences here. |
| T-06-V7-02 | Tampering | ANSI escape injection via user-controlled strings (e.g. baseUrl from YAML printed inside chalk.red) | mitigate | chalk wraps input as a literal string argument and prepends/appends escape codes — input cannot inject sequences that survive chalk's literal treatment (verified at chalk source per 06-RESEARCH §Security Domain row 2). No additional sanitization needed. |
| T-06-V7-03 | Information Disclosure | err.message from ConfigError contains the absolute YAML file path (resolved via process.cwd()) | accept | Same as T-06-V7-01 — local-only CLI, the user's own working directory paths are not sensitive in this context. The path is also actionable for the user ("which config file failed?"). |
| T-06-V7-04 | Tampering | Non-Error throws bypass the type-dispatched formatter | mitigate | Task 2 adds a defensive guard at the top of formatError: `if (!(err instanceof Error)) return chalk.red('Unexpected error: ') + String(err);` — covers `throw 'string'`, `throw { msg: 'oops' }`, `throw null`, etc. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new packages introduced (per 06-RESEARCH §Package Legitimacy Audit). Same disposition as 06-01's T-06-SC row. |
</threat_model>
