<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local node_modules | Supply-chain boundary: `npm install` pulls code from registry.npmjs.org into the local project. Compromised or typo-squatted packages execute via postinstall hooks. |
| package.json → executable bin | Local boundary: the bin field tells npm to symlink `framershot` into PATH (after `npm link` or global install). A malicious index.js modification would run with user privileges. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC | Tampering | npm install (Task 2) | mitigate | All seven dependencies are mainstream registry packages with known maintainers (Microsoft, Lovell Fuller, TJ Holowaychuk, Vitaly Puzrin, Colin McDonnell, Sindre Sorhus). Task 2 explicitly enumerates the seven by name in `<action>` so the executor cannot silently substitute typo-squats (e.g., `playwrite-chromium`, `shrp`, `comander`). package-lock.json captures the resolved tree for reproducibility. |
| T-01-PIH | Tampering | Postinstall hooks (playwright-chromium downloads a browser binary) | accept | playwright-chromium's postinstall downloads Chromium from Microsoft's CDN — this is the package's documented behavior and required for Phase 3. Mitigation would mean disabling postinstall, which breaks the tool. Accepted because the package is the official Microsoft Playwright distribution. |
| T-01-EE | Elevation of Privilege | Bin entry + shebang | accept | The bin runs with the user's own privileges only after `npm link` or `npm install -g`, both of which are explicit user actions. No setuid, no system service. Personal-tool threat surface is the user's own shell. |
| T-01-I | Information Disclosure | package.json metadata | mitigate | `private: true` and absence of repository/author/keywords fields keep this manifest from being mistakenly published to npm or indexed publicly. PROJECT.md Out of Scope explicitly forbids npm publish. |

Note: No `[ASSUMED]`/`[SUS]`/`[SLOP]` package legitimacy gates required — RESEARCH.md was skipped because PROJECT.md's locked stack list contains only well-known mainstream packages. Task 2 enumerates them by exact name in the action prose so substitution is detectable on review.
